
# Brugergrænseflade og interaktionsdesign

## 1. UI-filosofi

UI er baseret på notebook-paradigmet. Arbejdet foregår lineært fra top til bund.

Dette matcher skriftlige besvarelser.

## 2. Bloktyper

### Tekstblokke
- forklaringer
- ræsonementer
- konklusioner
- kan referere til variable
- formatte er endnu ikke bestemt men måske baseret på md og latex; dog vil dette kræve hjælp til eleverne fra præsentationslaget.

### Matematikblokke
- CAS-input
- parses og beregnes
- returnerer strukturerede resultater 

### Grafikblokke
- funktionsplot
- punktplot
- parametergrafer
- regression
- statistiske diagrammer
- grafik fra filer/clipboard? eller i gene blokke?

### Datablokke
- tabeldata
- manuel indtastning
- import fra Excel og clipboard

## 3. Opgaveblokke (scope)

Dokumentet opdeles i opgaveblokke.
Hver opgaveblok har sit eget scope.

Pædagogisk svarer det til "nyt ark papir"/en ny matematisk opgave.

## 4. Grafik (overordnet)

Grafikblokke er knyttet til matematiske definitioner
og opdateres reaktivt.

Grafikken skal understøtte:
- akser
- labels
- vinduevalg
- print-venlig visning
